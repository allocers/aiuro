#### **データ分析課題の整理**

##### **課題1：d払い（ポイント還元キャンペーン）**

| 項目 | 内容 |
| :--- | :--- |
| **理想的な状態** | ユーザーごとのキャンペーン効果（ポイント還元率の上昇に対する決済額の増加効果）を正確に予測し、費用対効果が高いユーザーと還元率の組み合わせに絞って施策を実施することで、**ROI（投資対効果）が最大化されている**。 |
| **事業観点の問題** | どのユーザーにどの程度の還元率を適用すれば最も費用対効果が高いのか不明確。一律的な施策ではポイント費用がかさむ。 |
| **事業的なアプローチ** | ユーザー属性や行動履歴に基づく**ターゲティング施策**（還元率のパーソナライズ）。 |
| **技術的に実現する方策** | 異質性のある因果効果（Conditional Average Treatment Effect: CATE）の推定。 |
| **期待できること** | ターゲティング精度が向上し、**キャンペーン全体のROIが改善**する。 |
| **最も単純な分析設計** | キャンペーン参加者（高還元率）と不参加者（低還元率）の期間中における平均決済額を比較する。 |
| **必要最小限のデータ** | **分析単位**: ユーザー \<br\> **時期軸**: キャンペーン前後各1ヶ月 \<br\> **利用指標**: 決済金額、ポイント還元率 \<br\> **データ処理**: 参加者・不参加者それぞれの決済額の平均値を計算し、その差分を見る。 |
| **単純な設計の都合** | キャンペーンに元々積極的なユーザー（高利用が見込まれるユーザー）が参加しやすいという**セレクションバイアス**を考慮していない。 |

| **最も単純な分析設計** | 進呈されたボーナスポイント額が多いユーザー群と少ないユーザー群で、その後の積立継続率を比較する。 |
| **必要最小限のデータ** | **分析単位**: ユーザー \<br\> **時期軸**: 積立設定後6ヶ月間 \<br\> **利用指標**: 積立継続率、進呈ボーナスポイント額、月々の積立設定額 |
| **単純な設計の都合** | 元々投資意欲が高く、高額な積立を厭わないユーザーほど、高額なボーナスポイントがもらえる条件を満たしやすい（**セレクションバイアス**）。 |
| **懸念が課題解決を困難にするか** | **重大である。** 継続率の差がボーナス額の効果なのか、ユーザーの元々の投資意欲の違いなのかを区別できない。 |
| **洗練したアプローチ** | 進呈ボーナスポイント額を処置量（多値処置）と捉え、**Coarsened AIPW推定量**を適用する。ボーナス額を「100pt」「500pt」「1000pt」などのビンに分け、「100ptから500ptに増額した場合、積立継続率が何%向上するか」を推定する。これにより、**ユーザーを習慣化させる上で最も費用対効果の高いインセンティブ額**を特定できる。 |

##### **課題2：NISA（dカード積立の利用促進）**

| 項目 | 内容 |
| :--- | :--- |
| **理想的な状態** | 顧客の投資ポテンシャルや金融リテラシーに応じて、**他社クレカからの乗り換えや新規積立を促すのに最適なポイント還元率**を提示できている。 |
| **事業観点の問題** | 最適な還元率が不明。低すぎると効果がなく、高すぎると、元々dカードで積立予定だった顧客に過剰なコストを払うことになり、収益性を損なう。 |
| **事業的なアプローチ** | 顧客のdカード利用状況や資産背景に応じた、**NISA積立還元率のパーソナライズ**。 |
| **技術的に実現する方策** | 異質性のある因果効果（CATE）の推定。 |
| **期待できること** | **dカード経由のNISA積立額の最大化**と、ポイント費用の最適化。 |
| **最も単純な分析設計** | 高い還元率が適用されたユーザー群と、低い還元率のユーザー群で、平均積立設定額を比較する。 |
| **必要最小限のデータ** | **分析単位**: ユーザー \<br\> **時期軸**: NISA口座開設後の1年間 \<br\> **利用指標**: 月々の積立額、適用されたポイント還元率 |
| **単純な設計の都合** | 高額な積立が可能な高所得層のユーザーに、より高い還元率が適用される傾向がある場合、**セレクションバイアス**が生じる。 |
| **懸念が課題解決を困難にするか** | **重大である。** 積立額の差が、還元率のインセンティブ効果なのか、ユーザーの元々の資産状況の違いなのかを識別できない。 |
| **洗練したアプローチ** | ポイント還元率を処置量（多値処置）と見なし、**Coarsened AIPW推定量**を用いる。還元率を「0.5%」「1.1%」「1.5%」などのビンに分け、「1.1%から1.5%への引き上げが、月々の積立額に与える純粋な増加効果」を推定する。これにより、**新規の投資資金を最も効率的に呼び込める還元率**を特定できる。 |

##### **課題3：ポイント還元（d払い利用と投資の連携）**

| 項目 | 内容 |
| :--- | :--- |
| **理想的な状態** | 日常決済しかしないユーザー層を、**最も効率的な「投資用ポイント」の還元率**で、自然にポイント投資の世界へ誘導し、新たなアクティブ投資家へと育成できている。 |
| **事業観点の問題** | 決済額の何%を投資用ポイントとして還元すれば、ユーザーが実際にポイント投資を利用してくれるのか、その閾値が不明。 |
| **事業的なアプローチ** | ユーザーの決済規模やポイント投資への関心度に応じた、**投資用ポイント還元率の最適化**。 |
| **技術的に実現する方策** | 異質性のある因果効果（CATE）の推定。 |
| **期待できること** | **決済から投資への新たな顧客体験の創出**と、**ポイント投資のアクティブユーザー数の増加**。 |
| **最も単純な分析設計** | 投資用ポイントの還元率が高いユーザー群と低いユーザー群で、ポイント投資の利用開始率を比較する。 |
| **必要最小限のデータ** | **分析単位**: ユーザー \<br\> **時期軸**: キャンペーン期間中および終了後1ヶ月 \<br\> **利用指標**: ポイント投資の利用開始率、適用された投資用ポイント還元率 |
| **単純な設計の都合** | 高い還元率のキャンペーンには、元々投資など新しいサービスに関心が高いユーザーが集まりやすい（**セレクションバイアス**）。 |
| **懸念が課題解決を困難にするか** | **重大である。** 利用開始率の差が、還元率の効果なのか、ユーザーの元々の好奇心の強さの違いなのかを判断できない。 |
| **洗練したアプローチ** | 投資用ポイントの還元率を処置量（多値処置）と捉え、**Coarsened AIPW推定量**を適用する。還元率を「0.5%」「1.0%」「2.0%」などのビンに分け、「0.5%から1.0%に引き上げた場合、ポイント投資の利用開始率が何%向上するか」を推定する。これにより、**ユーザーの行動変容を促す（＝投資を始めさせる）上で、最も費用対効果の高い還元率**を特定できる。 |
